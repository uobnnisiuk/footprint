# DEC-0008: Encounter保証線（FG=MUST / BG=BE）

## 日付
2026-02-06

## 決定（SR）
Encounter（遭遇・近接交換）について、保証線を次の通り固定する。

- **MUST: Capture（ローカル永続）**
  - ネット断絶でも、LS/LKG等の証拠は端末内に永続化されることを必須とする。

- **MUST: Foreground Encounter（FGでの遭遇・交換）**
  - 観測者（通行人/救助者等）がアプリを前景で利用している条件では、LS生成/交換が成立する設計を必須とする。
  - 災害時の有用性はこのFG条件（救助・探索が前景で動ける）で担保する。

- **BE: Background Encounter（BGすれ違い）**
  - iOSを含む各OSのバックグラウンド制約・端末差により、BGでの即時遭遇（すれ違い数秒）を保証線に置かない。
  - 成立した場合は上積みとして扱う。

## 理由
- BGすれ違い即時保証を要件にすると、iOS制約のため設計が凍結しやすい。
- 一方で「災害時に役に立つ」価値は、FG条件（救助/探索が前景で動ける）＋ Capture（ネット断絶でも残る）で維持できる。
- 公平性は「MUSTの範囲でOS差が出ない」ように置く（BGはBE）。

## 影響
- 仕様・テスト計画は、FG条件での成立を中心に組み立てる（BGはBEとして観測/計測対象に留める）。
- 平時の常時位置追跡/常時アップロードへスライドしない（ネットは上積み）。
- 多経路オーケストレーション（Transport Scope）を主軸にしない（DEC-0006）。

## 関連
- DEC-0007: Observer-first（一次証拠はLS）
- DEC-0006: Transport Scope 不採用
- RISK: iOS BLE制約（BGすれ違いの不確実性）
