# DEC-0007: Observer-first（一次証拠はLS / 外部観測）

## 日付
2026-02-06

## 決定（SR）
**footprint の一次証拠（primary evidence）は LS（Last Seen / 第三者観測ログ）とする。**
- 被災者端末が操作不能・端末破損・電池切れでも成立する設計思想を採用する。
- LKG（Last Known Good / 被災者自己記録）は有益だが、一次証拠の前提には置かず「補助証拠」とする。

## 背景 / 問題
初期構想は「平時からすれ違い等で"あの人を最後にここで見た（LS）"を蓄積し、有事に帰納的に救助者へ提示する」だった。
しかし議論の中で「本人が"私は今までここにいました（LKG）"を共有する」思想に寄り、被災者端末の健全性を暗黙前提にしてしまうズレが生じた。
本DECはそのズレを是正し、一次証拠をLSへ戻す。

## 影響（何が変わるか / 変わらないか）
- 変わる: 仕様上の中心は「観測者が生成するLSイベントの蓄積/提示」になる。
- 変わらない:
  - 位置追跡アプリ（平時の常時トラッキング）にはしない。
  - ネットは上積みであり、ネット断絶でもローカルに証拠が残ることを要件とする。
  - OS特権網（Find My/Find Hub等）のタグ位置をfootprintが自動統合する前提は採用しない。

## 補助証拠の扱い（LKG）
- LKGは「被災者端末が健全で記録できた場合」に救助判断を補助する証拠として扱う。
- LSとLKGは別ストリームであり、暗号学的な一意紐付けを前提にせず、時刻/場所の相関（confidence）で統合表示する。

## 関連
- DEC-0006: Transport Scope 不採用（多経路オーケストレーションを主軸にしない）
- RISK: iOS BLE制約（操作ゼロのP2P中継をスマホ単体で保証できない）
