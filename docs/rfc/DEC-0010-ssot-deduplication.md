# DEC-0010: SSOT 文書の重複解消と構造整理

## 日付
2026-02-06

## 決定内容（SR）

**SSOT 文書間の重複を「定義元を1箇所に確定、他は参照」の原則で解消する。**

### 変更一覧

| # | 変更 | 対象ファイル |
|---|------|-------------|
| 1 | OPEN 追跡を `80_risks.md` に一本化 | `10_`, `15_`, `80_` |
| 2 | 全 IF 索引を `00_constitution.md` に追加 | `00_`, `15_` |
| 3 | 通知最小セットの定義元を `15_` に確定 | `20_` |
| 4 | Product/Outcome を DEC-0007 に整合 | `00_` |
| 5 | 「暫定まとめ」削除（他 SSOT と丸かぶり） | `80_` |
| 6 | 未作成 Related 参照削除 | `00_` |
| 7 | Encounter 保証線の重複簡素化 | `10_` |

### 原則

- **情報の定義は1箇所のみ**。他箇所は参照（ポインタ）に置換する
- 仕様・不変条件・OPEN の情報は一切削除しない（移動・集約のみ）

## 理由

1. **SSOT の信頼性**: 同一情報の複数箇所記載は更新漏れ→矛盾の原因
2. **メンテナンスコスト**: 1箇所更新で済む構造にすることで RFC/DEC 適用時の変更漏れを防ぐ
3. **可読性**: 全 IF 索引・OPEN 一元管理により、プロジェクト全体像の把握が容易になる
